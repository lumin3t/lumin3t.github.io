digital forensics and incident response

Basic concepts

Artifacts Artifacts are pieces of evidence that point to an activity performed on a system. Artifacts can be collected from the Endpoint or Server’s file system, memory, or network activity.  Windows Forensics 1, Windows Forensics 2, or the Linux Forensics

Windows systems are primarily used for endpoints and server use-cases, like Active Directory Domain Controllers or MS Exchange email servers. Enterprises primarily use Linux systems in the capacity of servers hosting some service, for example, web servers or database servers.

Incident Response process

Post-incident Activity Similarly, SANS has published an Incident Handler’s handbook. The handbook defines the steps as follows: 5. Preparation
6. Identification 7. Containment 8. Eradication 9. Recovery 10. Lessons Learned\
